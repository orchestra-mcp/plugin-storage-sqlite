# Settings Panel Redesign — Match Native macOS System Settings

## Problem\nThe Orchestra Settings panel is broken and doesn't match native macOS System Settings design. The sidebar styling, section layouts, and content width don't follow Apple's design patterns.\n\n## Current State\n- `apps/swift/Shared/Sources/Shared/Plugins/SettingsPlugin.swift` (1,568 lines)\n- Window created as `NSPanel` in `apps/swift/Apps/macOS/OrchestraApp.swift` (lines 268-283)\n- Uses custom `HStack` with manual sidebar (200pt fixed) + `Divider()` + detail ScrollView\n- Custom components: `SettingsGroup`, `GroupRow`, `GroupToggle`, `SectionLabel`\n- 8 sections: General, AI, Voice, Permissions, Notifications, Prompts, Actions, About\n\n## Design Issues vs Native macOS System Settings\n1. **Sidebar is custom VStack of Buttons** — should use native `List` with `.sidebar` style for proper selection, keyboard nav, and scrollbar behavior\n2. **Section icons use gradients** — macOS uses solid flat colors, no gradients\n3. **No keyboard navigation** — arrow keys don't switch sections in sidebar\n4. **Hero header scrolls away** — macOS pins the icon+title, only content below scrolls\n5. **Content not full-width** — grouped controls should stretch to fill available width like macOS\n6. **TextField styling inconsistent** — uses `.roundedBorder` everywhere instead of native `.plain` with custom borders\n7. **Search field styling basic** — needs native `SearchField` appearance\n8. **No search highlighting** — filters sections but doesn't highlight matched text\n9. **Version hardcoded** — About section shows \"0.1.0\" instead of reading from `Bundle.main.infoDictionary`\n10. **Sidebar selected state** — uses `.accentColor` background but doesn't match native List selection style\n\n## Requirements\n1. **Sidebar**: Replace custom VStack+Buttons with `List` using `.listStyle(.sidebar)` for native selection, keyboard nav, and scrollbar\n2. **Sidebar icons**: Solid flat colors (no `.gradient`), 22×22pt rounded rect badges matching Apple's style exactly\n3. **Sidebar selection**: Native `List` selection binding — automatic highlight, focus ring, keyboard up/down\n4. **Content layout**: Full-width grouped sections using `Form` or `VStack` that stretches to fill available space\n5. **Group styling**: Match macOS grouped controls — `Color(nsColor: .controlBackgroundColor)` background, 10pt corner radius, full-width within content margins\n6. **Row styling**: Each row spans full width — label on left, control (Toggle/Picker/TextField) on right, proper dividers between rows\n7. **Hero header**: Large icon (56pt) + title (20pt bold) + subtitle (12pt gray) — centered, pinned at top (doesn't scroll away)\n8. **Scroll behavior**: Only the settings groups below the header should scroll\n9. **TextField styling**: Use `.plain` text field style with subtle borders matching native appearance\n10. **Search**: Native-feeling search with result highlighting\n11. **Keyboard navigation**: Full arrow key support for sidebar + Tab between controls\n12. **Dynamic version**: Read version from `Bundle.main.infoDictionary[\"CFBundleShortVersionString\"]`\n13. **Window**: Keep NSPanel but ensure proper sizing — min 680×480, ideal 780×600\n14. **Animations**: Smooth crossfade when switching sections (`.animation(.easeInOut(duration: 0.15))`)\n\n## Reference\nNative macOS System Settings (Settings.app) — see user screenshots showing:\n- Clean sidebar with flat-colored icons\n- Full-width content area with grouped rows\n- Proper spacing and typography\n- Native List selection behavior\n\n## Affected Files\n- `apps/swift/Shared/Sources/Shared/Plugins/SettingsPlugin.swift` (major rewrite of layout)\n- `apps/swift/Apps/macOS/OrchestraApp.swift` (window creation)
